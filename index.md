# 大作业要求
## 编码要求
1. 需要有交互窗口（可以是前后端、终端），可通过交互实现增删改查功能
2. 至少有5张表，3个有关联关系的实体
3. 必须有查询操作，并且有筛选、排序等基本功能
4. 必须有增删改操作
5. 不得有界面重复情况，待各组提交完选题情况后，会公布选题，如果有题目冲突请关注，重复作品对修改代码有较高要求，修改不好分数可能极低。

## 实验报告要求
1. 必须写明组长、组员及分工比例，工作量最低5%最多不超过60%（2人组队不超过70%）
2. 必须输出ER图
3. 必须输出数据表及每个字段的定义
4. 必须输出每个增删改查功能的SQL语句及对应交互界面的截图
5. 如果有事务设计，需在实验报告中【单列章节】写明哪里使用事务，为什么需要事务，如果不添加可能导致什么问题

## 过程性考核
1. 实验期间可能会抽查部分同学，如果进度较慢但期末检查完成度突飞猛进，可能会对修改代码有较高要求
2. 主动要求过程性检查，会提出新的扩展要求，但有加分

## 答辩要求
1. 能够介绍系统与数据库操作相关的功能及实现
2. 能够指出界面或操作对应的代码实现

## 高分要求（80以上）
1. 实现GROUP等方式实现统计功能
2. 实现分页查询功能（后端分页）
3. 界面美观，交互合理，需有WEB、APP或桌面应用作为前端载体
4. 事务支持
5. 实验报告内容丰富，包括界面截图、业务流程图、组件关系、SQL语句等。
6. 答辩流畅，能够快速指出代码实现位置，能够现场修改代码

## 大作业提交
1. 需提交电子版实验报告
2. 需要提交实验代码
3. 一份单独的包含组员姓名和分工的txt文件（和实验报告中一样，但单独提供一份）
4. 最终提交文件将实验报告、代码、分工情况打包提交，命名格式暂定为组号_系统名称_组长名_组长学号.zip
5. 最终提交的文件必须删除运行生成的bin或编译文件，只保留可编译运行的项目源码，压缩包需控制在20M以下。
